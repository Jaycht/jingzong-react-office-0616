## 二、已知问题（V2.13.0 待修复）

### 问题 4：深色模式编辑页白色背景
**根因**: DrawerNewRecord.tsx 及其子组件大量使用硬编码颜色内联样式：
- 步骤指示器: `#F6F8FB`, `#D8E1EA`, `#172033`
- repeatable section 容器: `#E2E8F0`, `#FAFBFC`
- DynamicField section 标题: `#D8E1EA`, `#155A8A`
- GlobalSuspectField 下拉菜单: `#fff`, `#E5E7EB`

CSS 变量只影响了 antd 组件和部分自定义样式，这些内联样式完全绕过了 CSS 变量系统。

### 问题 5：查看页面按钮点击区域异常
**根因**: CaseDetail.tsx 中的按钮是原生 `<button>`，在 `position: fixed` 全屏覆盖层内。SVG 图标可能阻挡文字区域的点击，或者 antd ConfigProvider 的深色主题注入了影响原生按钮的 CSS。

### 问题 6：全局联动性验证
**状态**: saveMassRecord/updateMassRecord 已自动重建索引，但需要验证：
- 批量导入（Excel）后索引是否重建
- getAllCaseNames 在时间轴中的实时性
- 所有数据写入路径是否都触发了索引重建

## 四、架构决策记录

### 存储架构
- massStore: IndexedDB（写穿缓存）
- inputHistoryStore: localStorage

## 七、开发教训

1. **不要做表面修复** — 应先找到根因再修复，而不是反复打补丁
2. **重复代码是 bug 温床** — MODULE_NAMES、rebuildSuspectIndex 等多处重复定义导致遗漏
3. **repeatable section 数据路径不一致** — 不同模块用不同的 key（suspects/coerciveMeasures），扫描逻辑必须覆盖所有可能
4. **CSS 变量 vs 内联样式** — 内联样式优先级高于 CSS 变量，深色模式必须用 JS 条件判断或 CSS 类
5. **npm ci vs npm install** — package-lock.json 与 package.json 版本不匹配时 npm ci 会失败
6. **版本号必须同步更新** — version.ts + package.json + CHANGELOG 三处必须同时更新
