## 第四阶段：根因分析 (V2.13.0)

### 外部根因分析
用户提供了一份详细的根因分析报告，指出：

1. **React 版本误判** — 项目已经是 React 18.3.1，不是 React 19
2. **数据路径不匹配** — CaseGraph/rebuildSuspectIndex/recordFormFields 只扫描顶层字段，遗漏了 coerciveMeasures repeatable section 中的嫌疑人数据
3. **索引重建不完整** — saveMassRecord/updateMassRecord 没有自动调用索引重建
4. **深色模式硬编码** — DrawerNewRecord 内联样式绕过CSS变量
5. **按钮点击区域** — SVG图标或flex布局干扰点击

### AI 修复措施

**修复1: CaseGraph扫描coerciveMeasures**
```typescript
// 增加了对 coerciveMeasures 数组的扫描
if (Array.isArray(d.coerciveMeasures)) {
  for (const item of d.coerciveMeasures) {
    const name = String(item?.suspect || item?.suspectName || '').trim();
    if (name) suspectNames.add(name);
  }
}
```

**修复2: rebuildSuspectIndex支持所有repeatable section**
```typescript
// 增加了对 coerciveMeasures 和其他 repeatable section 的扫描
const coercive = data.coerciveMeasures;
if (Array.isArray(coercive)) {
  for (const c of coercive) {
    const cname = typeof c.suspect === 'string' ? c.suspect.trim() : '';
    if (!cname) continue;
    map[cname] = { ... };
  }
}
```

**修复3: saveMassRecord/updateMassRecord自动重建索引**
```typescript
// 在保存记录后自动重建索引
records.unshift(record);
indexedDBAdapter.setItem(STORAGE_KEY, records);
rebuildCaseIndex(records);
rebuildSuspectIndex(records);
```

**修复4: recordFormFields递归处理repeatable section**
```typescript
// 递归扫描所有 repeatable section 数组
for (const key of Object.keys(data)) {
  const val = data[key];
  if (Array.isArray(val) && val.length > 0 && typeof val[0] === 'object') {
    for (const item of val) {
      if (typeof item !== 'object' || item === null) continue;
      for (const k of Object.keys(item)) {
        if (SKIP_KEYS.has(k)) continue;
        const v = item[k];
        if (typeof v === 'string' && v.trim()) {
          recordFieldValue(k, v);
        }
      }
    }
  }
}
```

## 第五阶段：版本管理教训

### 安装包文件名问题
用户发现安装包名称一直定格在 V2.12.8，没有随版本号更新。

**根因**: package.json 中的 version 字段没有同步更新。

**教训**: 每次提交必须同时更新：
- `src/version.ts` (APP_VERSION)
- `package.json` (version)
- CHANGELOG 条目

## 开发教训总结

1. **不要做表面修复** — 应先找到根因再修复，而不是反复打补丁
2. **重复代码是bug温床** — 多处重复定义导致遗漏
3. **repeatable section数据路径不一致** — 不同模块用不同key，扫描逻辑必须覆盖所有可能
4. **CSS变量vs内联样式** — 内联样式优先级高于CSS变量
5. **npm ci vs npm install** — package-lock.json不匹配时npm ci会失败
6. **版本号必须同步更新** — version.ts + package.json + CHANGELOG三处必须同时更新
7. **用户反馈要及时响应** — 不要让用户反复测试同一个问题
8. **代码审查要彻底** — 不要只看表面，要深入理解数据流
