### 严重 bug 修复：网页首屏死循环白屏（Maximum update depth，V2.14.19）
- 现象：用户每次网页（dev 127.0.0.1:5173 或 CloudStudio）**第一次打开**就报 "页面出现异常，请刷新后重试" + `Maximum update depth exceeded`，栈指向 `AppLayout.tsx:90` → `setCurrentPage(sessionSlice.ts:49)`。控制台另见 antd `destroyOnClose` 弃用、react-router v7 future-flag 警告。
- 根因（jsdom 埋点实证）：`AppLayout.tsx` 原有**两个双向同步 effect**——(A) `URL→store`：`urlPage!==currentPage` 时 `setCurrentPage(urlPage)`；(B) `store→URL`：`urlPage!==currentPage` 时 `navigate("/app/"+currentPage)`。挂载时二者**同拍触发且快照不一致**：A 把 store 设成"URL 派生值"，B 把 URL 导航到"/app/+store 值"，形成 `/app/dashboard ↔ /app//` 二态死循环（双斜杠让 strip 出来的值在 `dashboard` 与 `/` 之间反复横跳）。
- 修复：改为**单向同步**——`store.currentPage` 为唯一事实源，URL 仅作镜像（effect 依赖 `[currentPage, location.pathname, navigate]`，仅当 `urlPage!==currentPage` 时 `navigate`）；**外部导航**（浏览器后退/前进、书签、手动改地址、刷新直达）改用 `hashchange` 监听器单向回写 store（挂载时先 `applyFromUrl()` 采纳 URL，再 `store→URL` 同步，确保声明顺序让 URL 优先，避免被默认页覆盖）。并删除 AppLayout 中已无引用的 `setCurrentPage` selector（改用 `useAppStore.getState().setCurrentPage`）。
- 顺带清掉用户看到的另外两个控制台告警：`destroyOnClose`→`destroyOnHidden`（ProfileModal/PasswordModal/ModulePage 共 3 处）；`HashRouter` 加 `future={{ v7_startTransition:true, v7_relativeSplatPath:true }}`。
- 验证流程：先用 4 个复现用例（含 `/app/dashboard`+`dashboard` 这种"看似应收敛却仍循环"的极简用例）在 jsdom 真实挂载 `AppLayout` 复现 `Maximum update depth`（确认根因），修复后 7 个复现用例全绿；新增回归守卫 `src/__tests__/smoke.applayout.test.tsx`（MemoryRouter + 4 路径/页面组合，`not.toThrow`）。最终：`npx tsc -p tsconfig.app.json --noEmit --skipLibCheck` 0 错误；`npm run test` 27/27 全过；`npm run build` 通过；CloudStudio 重新部署同一链接。版本号按发布规范 → V2.14.19，CHANGELOG 顶部 2 条（死循环修复 + 告警清理）。

## 23:14 圆角真正修复（V2.14.24）
- 背景：V2.14.23 用 `Modal.styles.root` 设 border-radius+overflow:hidden，但 antd v6 的 `styles.root` 作用的是全屏 `.ant-modal-root`，不是白色卡片 `.ant-modal-content`，故底边仍是直角。
- 修复：ProfileModal 加 `rootClassName="profile-modal-root"`，并在 `src/index.css` 末尾加规则 `.profile-modal-root .ant-modal-content { border-radius:16px !important; overflow:hidden !important; }` 直接命中卡片；移除无效的 `root`/`wrapper` 圆角 styles。
- 验证：tsc 0 错误；27/27 测试；build 通过；CloudStudio 同一链接部署。版本 → V2.14.24。
- 经验：`antd v6 Modal` 圆角/裁切类样式不要用 `styles.root`（命中全屏根），应针对 `.ant-modal-content`（用 className + 全局 CSS 最稳，不依赖 styles 的 TS 键名）。
- ⚠️ 重要勘误：antd v6 的白色卡片真实类名是 `.ant-modal-container`（见 node_modules/antd/es/modal/style/index.js 第122行 `[componentCls]-container`），**`.ant-modal-content` 根本不存在**。V2.14.24 把 CSS 加在 `.ant-modal-content` 上完全无效，用户强制刷新仍「上圆下直」。V2.14.25 改为 `.profile-modal-root .ant-modal-container` 仍不生效——user 用无痕+强刷确认部署是最新的 V2.14.25，说明 antd 运行时注入的 container 样式（hashPriority:high）在级联上压过我们的规则，光改 container 类无法稳定生效。
- ✅ 终极方案（V2.14.26，已验证）：用 `Modal` 的 `modalRender={(node)=><div className="profile-modal-shell">{node}</div>}` 把内容包进自管外壳；全局 CSS 中 `.profile-modal-shell{圆角16+overflow:hidden+白底+阴影}` 完全接管卡片外观，并用 `.profile-modal-root .ant-modal-container{background:transparent!important;border-radius:0!important;overflow:visible!important;box-shadow:none!important;padding:0!important}` 把 antd 原生 container 透明化中和。注意 jsdom 实测：`.profile-modal-shell` 是 `.ant-modal-container` 的【父】节点（modalRender 在外层），所以 neutralize 规则用 `.profile-modal-root .ant-modal-container` 仍命中。此方案不依赖 antd 任何类名级联，确定生效。
