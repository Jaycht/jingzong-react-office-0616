## V2.21.4 打印表格根因修复（提交 a9715e0）
- **背景**：V2.21.3 仅改 CSS，但 antd 的 `ellipsis`（td 内包裹带 `white-space:nowrap;text-overflow:ellipsis` 内联样式的 span）与 `fixed` 列（生成 `position:sticky` + 分离式 `.ant-table-header`/`.ant-table-body` 双表格）由 JS 注入**内联样式**，CSS `!important` 覆盖不住，编号表头仍错位、内容仍截断。

## V2.21.5 修复进入模块页 TDZ 崩溃（提交 e478621）
- **根因**：V2.21.4 给 `dynamicColumns` 的 `useMemo` 增加 `printing` 依赖并在工厂内引用 `printing`，但 `const [printing, setPrinting] = useState(false)` 声明在第 515 行、晚于第 302 行的 `useMemo`。渲染时 `useMemo` 先执行，闭包/依赖数组访问到尚未初始化的 `printing`，抛 `ReferenceError: Cannot access 'it' before initialization`（TDZ），导致进入任何模块页直接崩溃（ErrorBoundary 捕获）。
- **修复**：将 `printing` 状态声明上移至 `dynamicColumns` 之前（第 301 行后），删除原第 515 行的重复声明；`printStyleRef`/`handlePrintList`/`useEffect` 保持原位不受影响。
- **教训（重要）**：在组件函数体内，若 `useMemo`/`useEffect` 的工厂或依赖数组引用了某个 `useState`/`const`，**必须保证该状态/变量在 `useMemo` 之前声明**。TDZ 错误 `Cannot access 'X' before initialization` 在 TS 编译与 `npm run build` 阶段均不报错，只在运行期首次渲染触发，务必在改完依赖后实际进页面验证。
- **版本与验证**：V2.21.5（src/version.ts + package.json 同步，CHANGELOG 首条）。`npm run typecheck` 0 错误；`npm run build` 成功；`npm run test` 27/27 全过。CloudStudio 同链接部署成功。提交 e478621（3 文件 +8/−4）。`.workbuddy/` 未提交。
